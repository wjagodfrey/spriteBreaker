Workflow guide

  1. Load a fresh spritesheet using the  New button
  2. Add and name a new sprite eg "mario"
  3. Add and name a new action eg "stand"
  4. Add a new frame (shortcut: f )
  5. Set your view area
    • Scroll and click/drag on the small canvas to set the view area
  6. Set frame coordinates
    • Left click/drag on the large canvas to set the top left frame coordinates
    • Right click/drag on the large canvas to set the bottom right frame coordinates
  7. Save your work with the  Save button
  8. Copy the resulting JSON from the  Output area into your project

Good to know

  • Change the JSON output fomat in the options dialog (the button)
  • Load and delete saved spritesheets with the  Load dialog
  • Update your current spritesheet with the  Update button. For browser security reasons you'll need to point the app to your spritesheet at least once each time you start the application, but don't worry. It's easy.
Because SpriteBreaker makes use of the latest web standards, people using older browsers may run into problems. Then again, I'm not really sure why anyone with an outdated browser would be using this.
SpriteBreaker is build using CanvasQuery by @rezoner, AngularJS and JQuery
